# 💣 Minesweeper (версія 0.2 — DOM інтеграція)

## 🔍 АНАЛІЗ КОДУ

### 1) HTML (index.html):
* **Підключення стилів** реалізовано у `<head>` за допомогою тегу `<link rel="stylesheet" href="styles.css">`.
* **Адаптивність:** сайт підтримує масштабування до розміру екрану через `<meta name="viewport" content="width=device-width, initial-scale=1.0">`.
* **Структура `<body>`:** реалізовано ієрархію контейнерів: батьківський `game-container` та вкладені `game-header` і `game-board`.
* **Елементи керування:** у `game-header` розміщено лічильник прапорців, ігровий таймер та кнопку *Reset game*.
    * Додано ідентифікатори (`id`) для швидкого доступу через JavaScript.
    * Додано підтримку **accessibility** через атрибут `aria-label`.
* **Ігрове поле:** статичні клітинки видалено. Контейнер `game-board` тепер слугує лише обгорткою, яка динамічно заповнюється через JavaScript.

### 2) CSS (styles.css):
* **Глобальні налаштування:** використано `box-sizing: border-box;` для коректного розрахунку розмірів.
* **Стилізація `<body>`:** колір фону `#008080` (класичний Windows 95), центрування контенту через `display: flex` та `min-height: 100dvh`.
* **Ігрова сітка:** побудована на **CSS Grid**. Розміри сітки (`grid-template-columns` та `rows`) тепер встановлюються динамічно (inline-стилями) з JavaScript, що робить гру гнучкою до зміни розміру поля.
* **Стани клітинок:** класи `.open`, `.mine-flagged`, `.exploded` динамічно додаються та видаляються скриптом залежно від ігрової ситуації.

### 3) JavaScript (script.js) — Логіка гри та DOM:
* **Архітектура даних:** Збережено глобальний об'єкт стану `gameState` та константи.
* **Рендеринг (DOM Manipulation):** * Функція `renderBoard()` динамічно створює HTML-структуру (елементи `<div>` з класом `cell`) на основі заданих розмірів.
    * Функція `updateBoardDOM()` синхронізує візуальний стан кожної клітинки з логічним масивом `field` (додає текст, цифри, емодзі мін та класи кольорів).
* **Обробка подій (Event Listeners):**
    * Використано **делегування подій**: обробники кліків вішаються на батьківський контейнер `game-board`, а не на кожну клітинку окремо.
    * **Лівий клік (`click`):** відкриває клітинку. Відбувається перевірка на поразку та рекурсивне відкриття `openCell()`.
    * **Правий клік (`contextmenu`):** викликає `toggleFlag()`. Стандартне меню браузера блокується за допомогою `e.preventDefault()`.
* **Управління інтерфейсом:**
    * Додано логіку ігрового таймера через `setInterval()`. Таймер стартує при першому кліці.
    * Додано лічильник доступних прапорців, який не дозволяє поставити прапорців більше, ніж кількість мін.
    * Зміна емодзі на кнопці рестарту залежно від стану гри.
* **Завершення гри:** При перемозі всі невідкриті міни автоматично позначаються прапорцями. При поразці — показується розташування всіх мін (і виділяється червоним та, на якій підірвався гравець). Візуальні сповіщення реалізовані з мікро-затримкою `setTimeout`, щоб браузер встиг відрендерити останні зміни перед блокуючим `alert`.
