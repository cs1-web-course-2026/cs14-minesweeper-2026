# 💣 Minesweeper (версія 0.2 — DOM інтеграція)

## 🔍 АНАЛІЗ КОДУ

### 1) HTML (index.html):
* **Підключення стилів** реалізовано у `<head>` за допомогою тегу `<link rel="stylesheet" href="styles.css">`.
* **Адаптивність:** сайт підтримує масштабування до розміру екрану через `<meta name="viewport" content="width=device-width, initial-scale=1.0">`.
* **Структура:** використано семантичний тег `<main>` як головний контейнер (`game-container`).
* **Елементи керування:** у `game-header` розміщено лічильник прапорців, ігровий таймер та кнопку *Reset game* (з атрибутом `type="button"`).
    * Додано підтримку **accessibility** (доступності) через атрибути `aria-label` для кнопок.
* **Доступність повідомлень:** Додано спеціальний текстовий контейнер `<p role="status" aria-live="polite">` для озвучення статусу гри скрінрідерами.
* **Ігрове поле:** статичні клітинки видалено. Контейнер `game-board` динамічно заповнюється через JavaScript.

### 2) CSS (styles.css):
* **Глобальні налаштування:** використано `box-sizing: border-box;` для коректного розрахунку розмірів.
* **Стилізація `<body>`:** колір фону `#008080` (класичний Windows 95), центрування контенту через `display: flex` та `min-height: 100dvh`.
* **Ігрова сітка:** побудована на **CSS Grid**. Розміри сітки (`grid-template-columns` та `rows`) встановлюються динамічно (inline-стилями) з JavaScript.
* **Стани клітинок:** класи `.open`, `.mine-flagged`, `.exploded` динамічно додаються та видаляються скриптом залежно від ігрової ситуації.

### 3) JavaScript (script.js) — Логіка гри та DOM:
* **Архітектура даних:** Збережено глобальний об'єкт стану `gameState` (включаючи `availableFlags`) та константи.
* **Рендеринг (DOM Manipulation):** * Функція `renderBoard()` динамічно створює HTML-структуру (інтерактивні елементи `<button type="button">` з класом `cell`).
    * Функція `updateBoardDOM()` синхронізує візуальний стан кожної клітинки з масивом `field` та динамічно генерує детальні `aria-label` для підтримки доступності.
* **Обробка подій (Event Listeners):**
    * Використано **делегування подій**: обробники кліків вішаються на батьківський контейнер `game-board`.
    * **Лівий клік (`click`):** відкриває клітинку. Відбувається перевірка на поразку та рекурсивне відкриття `openCell()`.
    * **Правий клік (`contextmenu`):** викликає `toggleFlag()`. Стандартне меню браузера блокується за допомогою `e.preventDefault()`.
* **Управління інтерфейсом:**
    * Додано логіку ігрового таймера через `setInterval()`.
    * Додано лічильник доступних прапорців, який не дозволяє поставити більше прапорців, ніж кількість мін.
* **Завершення гри:** При перемозі всі невідкриті міни автоматично позначаються прапорцями. Сповіщення про результати гри виводяться текстом у спеціальний DOM-елемент без використання блокуючих `alert()`.
