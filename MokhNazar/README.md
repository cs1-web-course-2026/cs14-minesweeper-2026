# 💣 Minesweeper (версія 0.1)

## 🔍 АНАЛІЗ КОДУ

### 1) HTML (index.html):
*   **Підключення стилів** реалізовано у `<head>` за допомогою тегу `<link rel="stylesheet" href="styles.css">`.
*   **Адаптивність:** сайт підтримує масштабування до розміру екрану через `<meta name="viewport" content="width=device-width, initial-scale=1.0">`.
*   **Структура `<body>`:** реалізовано ієрархію контейнерів: батьківський `game-container` та вкладені `game-header` і `game-board`.
*   **Семантика:** контейнери реалізовані з використанням тегів `<div>` та класів. `game-header` реалізовано через тег `<header>`.
*   **Елементи керування:** у `game-header` розміщено лічильник, таймер та кнопку *Reset game*.
    *   Для кнопки використано символ смайлика (стилістика Windows 95).
    *   Додано підтримку **accessibility** через атрибут `aria-label`.
*   **Ігрове поле:** контейнер `game-board` містить клітинки, розміщені за допомогою **CSS Grid**. На етапі макету клітинки вставлені вручну, хоча у фінальній версії вони будуть генеруватися через JavaScript.

### 2) CSS (styles.css):
*   **Глобальні налаштування:** використано `box-sizing: border-box;` для коректного розрахунку розмірів елементів з урахуванням меж.
*   **Стилізація `<body>`:**
    *   Колір фону `#008080` (класичний Windows 95).
    *   Центрування контенту через `display: flex`, `justify-content: center` та `align-items: center`.
    *   Використано `min-height: 100dvh` для коректного відображення на мобільних пристроях з адресним рядком.
*   **Game Container:** реалізовано ефект "випуклості" через `border: 3px solid` та специфічні кольори меж `#808080 #ffffff #ffffff #808080`.
*   **Розміри:** використано статичні розміри у пікселях для збереження пропорцій ретро-інтерфейсу на різних екранах.
*   **Header:** за допомогою `flexbox` та `justify-content: space-between` елементи (лічильники та кнопка) рознесені по краях та центру.
*   **Статистика та Кнопки:**
    *   `stat-box`: встановлено `font-size: 2rem` для великого шрифту.
    *   `reset-button`: додано `cursor: pointer` та анімацію натискання через псевдоклас `:active`.
*   **Ігрова сітка:** побудована на **CSS Grid**.
*   **Клітинки:** створено окремі класи для станів: `cell`, `mine`, `open`, `flag`, `exploded`, `mine-flagged`. `mine` та `flag` поки що є лише семантичними.
*   **Цифри:** класи від `mine-count-1` до `mine-count-5` задають специфічні кольори для цифр, що відповідають оригіналу.

### 3) JavaScript (script.js) — Логіка гри:
* **Архітектура даних (Data Layer):** Створено глобальний об'єкт стану `gameState` (зберігає розміри поля, кількість мін, поточний статус гри та таймер) та двовимірний масив `field` для представлення ігрової сітки.
* **Константи (Enums):** Щоб уникнути "магічних чисел" та "сирих" рядків, використано константи для конфігурації поля (`DEFAULT_ROWS`, `DEFAULT_COLS`, `DEFAULT_MINE_COUNT`), станів гри (`GAME_STATUS`), станів клітинок (`CELL_STATE`) та їх вмісту (`CELL_CONTENT`).
* **Генерація поля:** Функція `generateField()` ініціалізує сітку та випадковим чином (`Math.random()`) розставляє задану кількість мін, забезпечуючи захист від дублювання мін в одній клітинці та нескінченних циклів.
* **Бізнес-логіка (Алгоритми):**
    * Функція `countNeighbourMines()` розраховує кількість мін навколо кожної порожньої клітинки, використовуючи загальний масив зсувів `DIRECTIONS` (перевірка 8 сусідів).
    * Функція `openCell()` обробляє логіку відкриття клітинок: програш при потраплянні на міну (`GAME_STATUS.LOST`), а також **рекурсивне відкриття** сусідніх порожніх клітинок (метод Flood Fill), якщо поруч немає мін.
    * Функція `checkWinCondition()` перевіряє умову перемоги (всі безпечні клітинки успішно відкриті).
* **Інтерактив та Таймер:** Додано функцію `toggleFlag()` для керування прапорцями, а також функції `startTimer()` та `stopTimer()` для відліку ігрового часу через `setInterval`.
